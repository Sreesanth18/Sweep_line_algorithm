THE ALGORITHM

Objective: Given a set of line segments,compute all the intersection points of those line segments.

Structures Used

line_segment :- consists of 4 points (x1,y1,x2,y2) in the exact same order
event :- consist of two float points representing the x and y coordinates of the event
T_avl :- consists of a line segment,2 address to the left and right nodes and an integer representing the height
event_avl :- consists of an event two addresses to the left and right nodes and an integer representing the height

Functions Used

1. deleting a node (3.1 for event,3.2 for line_segment)
2. inserting a node(2.1 for event,2.2 for line_segment)
3. find intersection
4. add if upper

7. get the minimum value node
8. compare events
9. compare line_segments given event

15. get height
16. right rotate

18. find the line segment nearest and to the right side
19. find the line segment nearest and to the left side
20. find if intersected and add if the do in the event queue

TIMING ANALYSIS

Let there be n input line segments We have used balanced binary trees(AVL) to implement the event queue.Using the AVL trees each insertion and deletion take log (n) time. So to make the initial event queue nlog(n) time is required. Now for every additional intersection we find we spend O(log n) for its insertion and deletions So this brings our time complexity to O(nlog(n) + (I)log(n)) , where I is the number of intersections. The algorithm we use will tend to take advantage if there are fewer intersections as insertion and deletion take less than log(n) time and less I will lead to less time being taken.
